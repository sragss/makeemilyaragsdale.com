# MakeEmilyARagsdale.com — Project Context

Wedding website for Emily Devery & Sam Ragsdale. February 27, 2027. San Miguel de Allende, Mexico.

## Architecture

- **Next.js 16** App Router with Turbopack. All pages use Server Components by default.
- **Tailwind CSS v4** + **shadcn/ui** (new-york style, Radix primitives). Components in `src/components/ui/`.
- **Neon Postgres** via `@neondatabase/serverless` + **Drizzle ORM**. Schema in `src/db/schema.ts`, lazy DB init in `src/db/index.ts`.
- **Framer Motion** for animations (RSVP toggle, form transitions, botanical confetti).
- **NoiseBackground** component (`src/components/ui/noise-background.tsx`) — animated gradient + noise texture used for form inputs and toggle controls.

## Deployment

- **Vercel project**: `sragss-projects/makeemilyaragsdale-com`
- **GitHub**: `sragss/makeemilyaragsdale.com` — push to `main` auto-deploys to production

## Database

- **Neon project**: `makeemilyaragsdale` (org: Sam, region: aws-us-east-2)
- **Schema changes**: Edit `src/db/schema.ts`, then `pnpm db:push`
- **Seed data**: `pnpm db:seed` — creates 3 test invites with random pronounceable codes
- **Tables**:
  - `invites`: code, hotel_eligible, max_guests, notes, address, deleted
  - `guests`: name, is_primary, attending_friday, attending_saturday, email, phone, dietary_restrictions, plus_one_name
  - `hotel_bookings`: will_book, acknowledged_policy, booking_complete, booking_value
  - `invite_events`: type (view/belmond_click), ip, user_agent, created_at

## RSVP Flow

1. Guest visits `/rsvp` or `/rsvp/EBR-XXXXX` (direct link)
2. Enters pronounceable code (generated by `src/lib/codes.ts` — CVCVC pattern like TOVIK, BALEN)
3. URL updates to `/rsvp/CODE` via replaceState for persistence on refresh
4. Form collects: attendance toggle (coming/not coming), then per-event checkboxes (Friday pool party + Saturday wedding, both default checked), email, phone, dietary needs (pills), +1 name (solo invites only)
5. Hotel-eligible invites show Belmond booking section with images
6. Submit → botanical confetti if attending any event, "We'll miss you" if declining

## Admin

- `/admin` — password-protected (cookie-based, 30-day expiry, `ADMIN_PASSWORD` env var)
- Dashboard shows stats (attending/declined/pending/hotel bookings) + table of all invites
- Click any code → `/admin/rsvp/CODE` — edit all fields, mark hotel booking complete, set booking value
- "+ New Invite" button creates invites with auto-generated codes
- Activity log with IP geolocation map on each invite detail page

## Admin API

**All requests require `Authorization: Bearer <ADMIN_PASSWORD>` header.**

Base URL: `https://makeemilyaragsdale.com/api/admin` (or `http://localhost:3002/api/admin` for local dev)

### Reading Data

#### List all invites
```bash
curl -H "Authorization: Bearer emily-ragsdale" \
  "https://makeemilyaragsdale.com/api/admin?action=list"
```
Returns `{ invites: [...], count: N }`

#### Get single invite with events
```bash
curl -H "Authorization: Bearer emily-ragsdale" \
  "https://makeemilyaragsdale.com/api/admin?action=get&code=EBR-TOVIK"
```
Returns full invite with guests, hotel booking, and activity events (views, clicks).

#### Get stats
```bash
curl -H "Authorization: Bearer emily-ragsdale" \
  "https://makeemilyaragsdale.com/api/admin?action=stats"
```
Returns `{ invites, guests, attending, declined, pending, hotelEligible, hotelBooking, hotelComplete }`

### Writing Data

All writes are POST with JSON body. Every request must include `"action"` field.

#### Create invite
```bash
curl -X POST -H "Authorization: Bearer emily-ragsdale" \
  -H "Content-Type: application/json" \
  "https://makeemilyaragsdale.com/api/admin" \
  -d '{
    "action": "create",
    "guestNames": ["Jane Doe", "John Doe"],
    "hotelEligible": true,
    "address": "123 Main St, Austin TX 78701",
    "notes": "College friends"
  }'
```
- `guestNames` (required): array of strings, at least one
- `hotelEligible` (optional): boolean, default false
- `address` (optional): string, mailing address
- `notes` (optional): string, internal notes
- Returns `{ success, code, url, id }` — code is auto-generated (e.g. EBR-TOVIK)

#### Update invite settings
```bash
curl -X POST -H "Authorization: Bearer emily-ragsdale" \
  -H "Content-Type: application/json" \
  "https://makeemilyaragsdale.com/api/admin" \
  -d '{
    "action": "update_invite",
    "code": "EBR-TOVIK",
    "hotelEligible": false,
    "address": "456 Oak Ave, NYC 10001"
  }'
```
- `code` (required): invite code
- Optional fields: `hotelEligible`, `maxGuests`, `address`, `notes`

#### Update guest
```bash
curl -X POST -H "Authorization: Bearer emily-ragsdale" \
  -H "Content-Type: application/json" \
  "https://makeemilyaragsdale.com/api/admin" \
  -d '{
    "action": "update_guest",
    "code": "EBR-TOVIK",
    "guestName": "Jane Doe",
    "attendingFriday": true,
    "attendingSaturday": true,
    "email": "jane@example.com",
    "phone": "+1 555 123 4567"
  }'
```
- Identify guest by `guestId` OR by `code` + `guestName` (case-insensitive match)
- Optional fields: `name`, `attendingFriday` (true/false/null), `attendingSaturday` (true/false/null), `email`, `phone`, `dietaryRestrictions`, `plusOneName`
- Set `attendingFriday: null` / `attendingSaturday: null` to reset to pending
- Two events: Friday pool party (Feb 26) and Saturday wedding (Feb 27)

#### Update hotel booking
```bash
curl -X POST -H "Authorization: Bearer emily-ragsdale" \
  -H "Content-Type: application/json" \
  "https://makeemilyaragsdale.com/api/admin" \
  -d '{
    "action": "update_hotel",
    "code": "EBR-TOVIK",
    "willBook": true,
    "bookingComplete": true,
    "bookingValue": "4500.00"
  }'
```
- `code` (required): must be a hotel-eligible invite
- Optional fields: `willBook` (true/false/null), `bookingComplete` (boolean), `bookingValue` (string dollar amount)

#### Delete invite (soft delete)
```bash
curl -X POST -H "Authorization: Bearer emily-ragsdale" \
  -H "Content-Type: application/json" \
  "https://makeemilyaragsdale.com/api/admin" \
  -d '{
    "action": "delete",
    "code": "EBR-TOVIK",
    "confirm": true
  }'
```
- `code` (required): invite code
- `confirm` (required): must be `true` — safety check to prevent accidental deletion
- Soft deletes only (sets `deleted: true`). Data remains in DB and can be recovered.

### Error Handling

All errors return `{ error: "message" }` with appropriate HTTP status:
- 400: Bad request (missing fields, validation errors)
- 401: Missing or wrong Authorization header
- 404: Invite/guest not found

### Important Notes

- **Never modify the database directly.** Use this API or the admin UI at `/admin`.
- Invite codes are auto-generated pronounceable strings like `EBR-TOVIK`. Do not create custom codes.
- All codes are uppercase. The API normalizes input to uppercase automatically.
- Guest lookup by name is case-insensitive.
- Hotel operations only work on hotel-eligible invites. Set `hotelEligible: true` on the invite first.
- Delete requires `"confirm": true` as a safety check.
- `attendingFriday` and `attendingSaturday` each accept `true`, `false`, or `null` (pending/not yet responded).
- Two events: Friday pool party at the Belmond (Feb 26, 3-8 PM) and Saturday wedding at Luna Escondida (Feb 27).

## Design

- **Light mode only** — warm cream palette inspired by San Miguel de Allende (plaster walls, terracotta, sun-bleached stone)
- **Fonts**: Cormorant Garamond (serif, headings/toggle labels), Geist Sans (body), Geist Mono (codes)
- **Colors**: warm brown foreground (not black), muted terracotta primary, sand borders. All oklch in `globals.css`.
- **Design budget principle**: minimal visual treatments, spacing and typography do the hierarchy work
- **NoiseBackground**: animated warm gradients + noise texture on form inputs and toggle. Inset shadows for depth.
